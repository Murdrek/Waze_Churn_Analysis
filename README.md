# Waze_Churn_Analysis
User Churn Analysis: Waze Project
This project focuses on analyzing user driving behavior and engagement patterns to identify key factors leading to churn. The goal is to provide data-driven insights that help improve user retention for the Waze platform.

🔍 Project Overview
Client: Waze Data Team

Goal: Predict and analyze user churn through behavioral patterns.

Framework: PACE (Plan, Analyze, Construct, Execute).

Tools: Python (Pandas, NumPy), Statistical Analysis, Data Visualization.

Key Deliverables: - Exploratory Data Analysis (EDA) of user sessions.

Identification of "Super-Driver" churn profiles.

Statistical summaries of distance and driving frequency.

💡 Key Insights
Churn Profiling: Identified a distinct group of users (Super-Drivers) who drive significantly higher distances (~697 km/day) but show higher churn rates.

Data Integrity: Handled heavily right-skewed data using median-based analysis to ensure statistical accuracy.

📁 Repository Structure
/docs/ → Executive summaries and behavioral insights.

/notebooks/ → Data exploration, cleaning, and statistical testing.

/code/ → Python scripts for data processing pipelines.
